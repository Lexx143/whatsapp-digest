# WhatsApp Digest Bot → Telegram

*Read in English: [README.md](README.md). Полное описание архитектуры: [HANDOFF.md](HANDOFF.md).*

Автономный ежедневный аналитик для команд, живущих в WhatsApp. Каждое буднее утро он
читает локальную базу WhatsApp на Mac, расшифровывает голосовые (whisper), смотрит
картинки, анализирует переписку через Claude — с опорой на долговременную память о
клиентах, проектах и зависших задачах — и присылает структурированную сводку в
Telegram-группу. На сводку можно ответить прямо в Telegram — бот учтёт обратную связь
в следующих отчётах.

## Что присылает

- **Дневной** (пн–пт 09:30) — читается за 30 секунд: критическое, передачи между
  отделами, менеджеры одной строкой (клиент ✓ / внутреннее / риск), план/факт и
  эскалации задач, зависших N+ дней.
- **Недельный** (понедельник, вторым сообщением) — вехи, повторяющиеся заявки,
  динамика очередей и дисциплины, перенос на новую неделю.
- **Месячный** (первый понедельник, третьим сообщением) — вся динамика, которую
  позволяют данные (клиенты пришли/ушли, бэклог, план/факт-дисциплина по менеджерам,
  время жизни задач), плюс мета-анализ качества самих отчётов команды.

## Достоверность

Две защиты от «ИИ-слопа», добавленные после реальной критики команды:

1. **Маркировка «факт/вывод»** — всё, что бот вывел сам (а не прочитал), явно помечено
   в сводке.
2. **Контроль фактов** — перед каждой отправкой второй прогон Claude адверсарно сверяет
   каждое утверждение сводки с сырым транскриптом и удаляет/исправляет неподтверждённое.

## Как обучается

- `memory/context.md` — долговременная память, которую бот ведёт сам: клиенты, проекты,
  задачи с датами и статусами. Отсюда он знает, что задача «висит 14-й день».
- `memory/feedback.md` — любой участник Telegram-группы отвечает (reply) на сообщение
  бота или упоминает его; перед каждым прогоном бот забирает это и ставит указания
  людей выше собственных представлений о важном.

## Конвейер

`launchd (пн–пт 09:30)` → `digest.sh`:
`feedback.py` (сбор ответов из Telegram) → `extract.py` (SQLite WhatsApp → JSON, окно =
с прошлого успешного запуска) → `transcribe.py` (mlx-whisper, с кэшем) → `claude -p`
(анализ + обновление памяти) → контроль фактов → `tg.py send` → сдвиг `state/last_run`.
Пропущенные дни, выходные и выключенный Mac подхватываются без дублей. Ошибка любого
шага — сообщением в ту же Telegram-группу.

Код — в этом репозитории; все данные — вне git, в `~/.whatsapp-digest/`
(токены, память, архив сводок, state, логи, venv).

## Установка (~30 минут)

Требования: Mac с залогиненным WhatsApp Desktop (на macOS база чатов лежит в открытом
виде — на Windows нет), подписка Claude Code, ffmpeg, Python 3.11+.

1. Разово авторизовать Claude CLI для headless: `claude` → `/login` в Terminal.
2. Клонировать в **постоянную** папку, запустить `./install.sh` — создаст
   `~/.whatsapp-digest/`, venv с mlx-whisper (~1.5 ГБ модель при первом прогоне)
   и launchd-агент.
3. Вписать JID своих чатов в `~/.whatsapp-digest/chats.json` (`{"<jid>": "Название"}`) —
   как их найти, описано в [HANDOFF.md](HANDOFF.md).
4. Создать бота у @BotFather → токен в `~/.whatsapp-digest/config.env`; добавить бота
   в Telegram-группу, затем `python3 tg.py chat-id` → chat id в конфиг.
5. Тест: `./digest.sh` (за период: `./digest.sh --from 2026-07-01`; недельный вручную:
   `WA_WEEKLY=1 ./digest.sh`; месячный: `WA_MONTHLY=1 ./digest.sh`).

## Ограничения

- Mac должен быть включён, WhatsApp Desktop — запущен, иначе база не обновляется.
- Видео не расшифровываются (помечаются в транскрипте с подписями).
- Обратная связь применяется со следующего прогона; бот не отвечает в чате мгновенно.
- Пустое окно анализа — ничего не отправляется (осознанно).

Вся бизнес-логика — в четырёх промпт-файлах (`prompt.md`, `prompt-weekly.md`,
`prompt-monthly.md`, `prompt-verify.md`): адаптируйте их под свою команду, не трогая код.
